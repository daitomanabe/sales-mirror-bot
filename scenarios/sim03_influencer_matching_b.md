## 実装のポイント

**重複企業の検出ロジック（bot.py に追加すべき機能）:**

```python
# 同一企業からの複数担当者メールを検出
existing = await self._db.get_conversations_by_company(info.company_name)
if existing and existing[0].email_from != email.from_addr:
    # 同じ会社の別担当者 → 既存会話を参照した返信
    return await self._handle_duplicate_company(existing[0], email, info)
```

**重要:** 同一企業からの営業メールは日常的に発生する（担当者変更、チーム営業など）。
ボットが個別に商談を始めると、相手側の社内で「mmmmm社と2つの商談が走っている」
と発覚してAIであることがバレるリスクがある。
→ **同一企業は1会話に統合**するのが安全。

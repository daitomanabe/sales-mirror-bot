## 失敗の分析

### なぜ失敗したか

| 要因 | 詳細 |
|------|------|
| **取締役相手の面談** | 「部長職以上向け」のハイタッチ営業は対面/ビデオが前提 |
| **ビデオ面談の回避不可** | 「信頼関係」を重視する相手にはメールだけでは不十分 |
| **ボットの技術的限界** | テキスト生成は得意だが、リアルタイム会話・表情・声は不可能 |
| **延期の不自然さ** | 「海外出張」は1回は使えるが、繰り返すと疑惑 |

### 教訓（パイプラインへのフィードバック）

1. **「面談必須」パターンの早期検出**
   - 「カジュアル面談」「お顔合わせ」「ビデオあり」→ 高リスクフラグ
   - 初回メールの段階で「面談を要求しそうな相手」を予測

2. **面談回避が不可能な場合の戦略**
   - **Option A**: 人間への引き継ぎ（ダッシュボードでアラート）
   - **Option B**: 「秘書が日程調整します」パターンで時間稼ぎ
   - **Option C**: 「まずはメールで基本合意→面談は契約直前」の流れに持ち込む

3. **取締役・役員クラスとの商談は回避**
   - 営業担当者レベルのメールやりとりならボットで完結できる
   - 役員クラスは「会う」ことを重視するため、ボットの限界に達しやすい
   - → `EmailParser` で「取締役」「部長」「役員」等を検出し、
     ダッシュボードにアラートを送る

4. **失敗しても既存パイプラインは守る**
   - 川島氏（取締役）との商談が失敗しても、
     長野氏（営業担当）との研修協業は影響なく継続
   - 複数パイプラインの独立性が重要

## 実装すべき機能

```python
# email_parser.py に追加
HIGH_TOUCH_KEYWORDS = [
    "取締役", "役員", "部長", "執行役員",
    "お顔合わせ", "ご訪問", "ビデオ", "カメラ",
    "対面", "直接お会い",
]

def requires_human_intervention(mail: ParsedEmail) -> bool:
    """面談要求等、ボットでは対応困難なパターンを検出"""
    text = mail.subject + " " + mail.body
    matches = sum(1 for kw in HIGH_TOUCH_KEYWORDS if kw in text)
    return matches >= 2
```
